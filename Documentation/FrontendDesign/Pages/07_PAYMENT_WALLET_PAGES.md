## 11. State Management

### 11.1 Wallet State

```typescript
// Wallet balance query
const { data: wallet } = useQuery({
  queryKey: ['wallet', 'balance'],
  queryFn: () => api.wallet.getBalance(),
});

// Transactions query with pagination
const {
  data,
  fetchNextPage,
  hasNextPage,
} = useInfiniteQuery({
  queryKey: ['wallet', 'transactions', filters],
  queryFn: ({ pageParam = 0 }) =>
    api.wallet.getTransactions({ offset: pageParam, ...filters }),
  getNextPageParam: (lastPage) =>
    lastPage.hasMore ? lastPage.nextOffset : undefined,
});

// Purchase mutation
const purchaseMutation = useMutation({
  mutationFn: (data: PurchaseRequest) => api.wallet.purchase(data),
  onSuccess: () => {
    queryClient.invalidateQueries(['wallet', 'balance']);
    queryClient.invalidateQueries(['wallet', 'transactions']);
    toast.success('Purchase successful!');
  },
});
```

### 11.2 Subscription State

// Subscription status
const { data: subscription } = useQuery({

## 12. Security Considerations

### 12.1 Payment Security

| Aspect | Implementation |
|--------|----------------|
| Card handling | Stripe.js (PCI compliant) |
| Tokenization | Card details never hit our servers |
| 3D Secure | Enabled for high-risk transactions |
| HTTPS | All payment pages over SSL |
