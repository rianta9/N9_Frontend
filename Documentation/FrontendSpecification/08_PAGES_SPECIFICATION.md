#### 3.1.2 Component Specification

| Section | Components | Data Source |
|---------|------------|-------------|
| Hero | `HeroCarousel`, `FeaturedStoryCard` | `useFeaturedStories()` |
| Trending | `StoryCarousel`, `StoryCard` | `useTrendingStories()` |
| Recommended | `StoryCarousel`, `StoryCard` | `useRecommendedStories()` |

| New Releases | `StoryCarousel`, `StoryCard` | `useNewReleases()` |
| Completed | `StoryCarousel`, `StoryCard` | `useCompletedStories()` |

interface HomePageData {

// Prefetch on navigation
const loader = async () => {
  await Promise.all([
    queryClient.prefetchQuery(queryKeys.stories.featured()),
    queryClient.prefetchQuery(queryKeys.stories.trending()),
    queryClient.prefetchQuery(queryKeys.stories.newReleases()),

};

```typescript
interface PayoutRequestState {
  amount: number;
  method: 'paypal' | 'bank_transfer' | 'stripe';
  paypalEmail?: string;
  bankAccountId?: string;
  stripeAccountId?: string;
  isProcessing: boolean;
  error: string | null;
}

// Payout status aligned with backend
type PayoutStatus = 
  | 'PENDING'     // Request submitted
  | 'PROCESSING'  // Being processed
  | 'COMPLETED'   // Successfully paid
  | 'FAILED'      // Payment failed
  | 'CANCELLED';  // Cancelled by user/admin
```

## 8. Page Loading States

### 7.1 Loading Patterns

| State | Implementation | Description |
|-------|----------------|-------------|
| Initial Load | Skeleton screens | Show layout structure with animated placeholders |
| Navigation | Page transition | Subtle fade or slide animation |
| Data Refresh | Inline loading | Keep stale data, show refresh indicator |
| Infinite Scroll | Bottom loader | Spinner at bottom while loading more |
| Error State | Error component | Retry button with error message |
| Empty State | Empty illustration | Contextual message with CTA |
